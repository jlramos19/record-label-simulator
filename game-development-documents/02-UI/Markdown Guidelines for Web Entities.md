# Markdown Guidelines for Web Entities

## Purpose
Standardize how entity data is documented in the web app mirror. Replace engine terms with web data model terms.

## Terms
- Entity: a data record (not a scene object).
- UI component: a rendered view of entity data.
- Slot: a UI selection target bound to an entity ID.

## Example
- **Entity**: Community Chunk
- Fields: `Location`, `Population`, `Average Age`, `Preferred Themes`
- Values: shown inline or as tables for readability.
