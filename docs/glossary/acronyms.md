# Acronyms and Abbreviations

This glossary lists acronyms, initialisms, and shorthand used in canon docs and the web codebase.
If you add new shorthand, update this list.

## Product, roles, and systems
- RLS: Record Label Simulator.
- CCC: Creators Community Chamber.
- CEO: Chief Executive Officer (player role).
- GDD: Game Design Documents (folder name).
- MVP: Minimum Viable Product.
- AI: Artificial Intelligence (AI-controlled labels). (Confirm if another in-world expansion is intended.)
- ID: Identifier; used in "Creator ID" as the signed creator identifier. (Confirm if "ID" should be expanded differently in-world.)
- UI: User Interface.
- UX: User Experience.
- ECS: Entity-Component-System (see `docs/glossary/unity-to-web.md`).

## Charts, releases, and scheduling
- EP: Extended Play.
- LP: Long Play (album).
- DNC: Did Not Chart.
- LW: Last Week (chart position).
- WOC: Weeks on Chart.
- ASAP: As Soon As Possible.

## Tech, formats, and time
- ASCII: American Standard Code for Information Interchange.
- CSV: Comma-Separated Values.
- JSON: JavaScript Object Notation.
- NDJSON: Newline-Delimited JSON.
- HTML: HyperText Markup Language.
- CSS: Cascading Style Sheets.
- JS: JavaScript.
- TS: TypeScript.
- DOM: Document Object Model.
- URL: Uniform Resource Locator.
- UTC: Coordinated Universal Time.
- LTS: Long-Term Support.
- TBD: To Be Determined.
- AM/PM: Ante meridiem / Post meridiem.
- YYYY, MM, DD, YYYYMMDD, HHMMZ: Date/time format segments used in patch IDs (UTC).
- FM: Frequency Modulation (radio station shorthand).

## UI icon shorthand
- AR: Areas.
- CR: Creators.
- IT: Items & Tools.
- CO: Collabs.
